name: wjt-wakeup
description: 唤醒协议——长期搁置的项目重启时执行，按成本阶梯恢复，禁止跳级读源码


这是搁置已久的任务, 按阶梯恢复, 每级完成后判断"是否足够继续工作",
足够就停止恢复直接开工。禁止跳到L5:

L1 项目定位(10秒): 读 TASKS.md —— 本任务什么阶段, 是"没做完"还是"该废弃"
L2 找回"为什么"(1分钟): 读任务纪要 —— 优先「冷冻快照」区, 然后必读
   「踩坑墓地」(搁置前失败过的路, 醒来后最容易兴冲冲重走一遍)和「决定记录」
L3 找回"做到哪"(1分钟): git log --oneline --graph -15 && git status
   && git diff --stat; 对照「计划」区勾选状态
L4 定位精确断点(最准的一步): 在worktree内跑纪要中的「验收命令」——
   红着的测试就是断点本身。对红测试的报错先跑 aiwt when "<报错关键词>"
   检索历史病历。同时查影子快照:
   git for-each-ref --sort=-creatordate refs/snapshots/ | head -3
L5 只读断点相关源码: 仅打开「精确断点」指向的文件及直接依赖,
   禁止全库通读、禁止重新总结架构

恢复完成后: 「状态:」从 冻结 改回 编码(或测试), 「决定记录」追加
"<日期>: 解冻恢复", 从「重启第一步」开始干活。提醒人类跑 aiwt sync。
