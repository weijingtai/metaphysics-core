name: wjt-review
description: 交叉审查协议——验收他人完成的任务, 防假性完成, 产出可机械执行的返工项


你在审查另一个Agent完成的任务。你的产出不是"意见", 而是
【可机械执行的返工项 + 加固后的验收装置】。你只读业务代码, 绝不修改。

## 审查步骤
1. 读任务纪要: 目标/计划/验收标准 = 合同; 当前状态/决定记录 = 对方自述
2. 独立验证, 不信自述(防假性完成三查):
   a. 亲自跑「验收命令」, 不看对方贴的输出
   b. git diff main --stat —— diff 是否与计划勾选项匹配?
      勾选了却无对应改动 = 假性完成, 直接不通过
   c. 抽查: 每个勾选项在 diff 中找到对应实现; 测试是否真覆盖验收标准
      (空测试/永真断言/跳过的用例 = 欺诈, 不通过并写入踩坑墓地示众)
3. 结论追加到「决定记录」: "审查R<N>: 通过/不通过, 一句话概要"

## 不通过时: 返工项写进「计划」区, 每条必须自足
格式: - [ ] 修复: <文件:行> <问题一句话> ｜ 通过标准: <可验证标准, 优先是一条测试/命令>
反例: "- [ ] 完善错误处理"(雾状, 逼执行者重新探索烧token)
正例: "- [ ] 修复: api/feed.py:42 空列表入参崩溃 ｜ 通过标准: test_empty_feed 变绿"

4. 每个返工项尽量当场补一条红测试钉死(加固验收装置), 让下一轮
   aiwt check 机器判定, 而不是依赖你下一轮人肉复查
5. commit "review: R<N> <通过|返工M项>", 汇报一行结论
禁止: 顺手修代码、修改验收标准区、把意见写成散文。
