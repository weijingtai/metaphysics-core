description: 落盘进度到任务纪要（每完成一个子任务后执行, 不要等会话结束）
---

对 tasks/ 下当前分支对应的任务纪要执行标准落盘:

1. 提交代码: git add -A && git commit
   (完整功能 feat:/fix:/test: 前缀; 半成品 wip: 前缀并注明卡点)
2. 计划区: 勾选已完成项, 只改 checkbox 不重写任务描述
3. 「当前状态」节【覆盖重写】, ≤12行: 刚完成 / 半成品位置(文件:行) /
   下一步 / 微观意图(接下来2-3步的打算, 含"顺手要改的东西"这类
   只存在于你脑中的计划) / 验证方法
4. 「决定记录」: 本轮有技术选型或取舍则【追加】一行: 日期: 决定+理由
5. 「踩坑墓地」: 本轮有失败的尝试则必须【追加】入墓:
   日期: 尝试了X, 失败原因Y, 结论(别再试/换Z)。
   走不通的路和走得通的路同样珍贵, 不入墓=逼后任重踩一遍。
6. 纪要与代码一起 commit
7. 回复一行确认: "已落盘: <commit前7位> / 计划 <完成数>/<总数> / 新入墓 <N>条"

只写事实和决定, 不写过程叙述。禁止修改 TASKS.md 和「验收标准」区。
